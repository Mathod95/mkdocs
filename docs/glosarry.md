## A
### `Argo` 
: Argo est une suite d'outils open source pour Kubernetes, conçue pour gérer les workflows, les déploiements et l'automatisation. Elle est particulièrement utilisée dans les environnements DevOps et CI/CD natifs Kubernetes.
## B
### `Build`
: La phase de build, ou intégration, regroupe les actions qui transforment le code source en un livrable prêt à être déployé. Elle inclut l’analyse statique du code, l’exécution des tests unitaires, la gestion des dépendances et la compilation ou le packaging de l’application afin de produire un artifact exploitable.
## C
### `CRD`
: (Custom Resource Definition) est un élément Kubernetes qui permet d’ajouter de nouveaux types de ressources personnalisées au cluster afin d’étendre ses fonctionnalités sans modifier sa structure native.
### `CLI`
: (Command Line Interface) est une interface textuelle qui permet d’utiliser un logiciel ou un système en entrant des commandes dans un terminal.
### `CNI`
: (Container Network Interface) : Standard permettant de connecter les conteneurs à un réseau. Utilisé par Kubernetes pour gérer le réseau des pods via des plugins comme Cilium, Calico ou Flannel.

### `Helm`
: Helm est le gestionnaire de packages de Kubernetes ; il permet de déployer, configurer et mettre à jour des applications à l’aide de charts, des modèles regroupant l’ensemble des paramètres nécessaires, ce qui rend la gestion des configurations et des mises à jour cohérente et reproductible.

## K
### `KinD`
: (Kubernetes IN Docker) est un outil permettant de créer des clusters Kubernetes locaux à l’aide de conteneurs Docker. Il est principalement utilisé pour le développement, les tests CI/CD, ou l'expérimentation avec Kubernetes, sans avoir besoin de déployer un cluster sur le cloud.
## L
### `Load balancer`
: Un composant qui distribue le trafic réseau ou les demandes des utilisateurs entre plusieurs serveurs afin d’optimiser les performances, la disponibilité et la fiabilité d’une application.

## S
### `sailor.sh`
: Simulateur gratuit en ligne d’examens Kubernetes (CKA, CKAD, CKS) offrant des laboratoires pratiques et des scénarios réalistes pour s’entraîner.
### `SLA` 
: (Service Level Agreement) Contrat définissant les niveaux de service attendus entre un fournisseur et un client.
### `SLO` 
: (Service Level Objective) Objectif mesurable lié à la performance d’un service, utilisé pour évaluer le respect d’un SLA, par exemple une disponibilité de 99,9 %.
### `SLI` 
: (Service Level Indicator) Indicateur qui mesure la performance réelle d’un service par rapport à un objectif défini (SLO), comme le temps de réponse ou le taux d’erreur.
